# Noema

Сервис для создания и ведения вымышленных миров: таймлайны, карточные истории, бестиарий, биографии, карты и доски связей. Ориентирован на писателей, геймдизайнеров и сценаристов.

## Стек

- Laravel 12, PHP 8.2+
- SQLite (по умолчанию), совместимо с MySQL/PostgreSQL
- Vite, Tailwind CSS 4, DaisyUI

## Установка

```bash
composer install
cp .env.example .env
php artisan key:generate
php artisan migrate
npm install
npm run build
```

## Запуск

```bash
php artisan serve
```

Приложение: `http://localhost:8000`

## Архитектура данных и модули

Все сценарные модули привязаны к **миру** (`worlds`): у мира есть владелец (`user_id`), название, описание (синопсис), **точка отсчёта** (текстовая метка нуля для шкалы времени), опциональное изображение, флаг видимости (`onoff`).

### Мир и дашборд

- Список миров, создание мира, вход в дашборд мира.
- **Настройки мира** (модальное окно на дашборде): название, описание, точка отсчёта, изображение; сохранение через `PUT /worlds/{world}`.
- При создании мира автоматически поднимается **скелет таймлайна** (см. ниже).

### Таймлайн

- У каждого мира один набор **линий времени** и **событий** на них.
- При создании мира `TimelineBootstrapService` создаёт главную линию («История мира») и событие в **год 0** с заголовком из **точки отсчёта** мира (или строка «Точка отсчёта», если поле пустое). Это событие помечено флагом `is_reference_marker`.
- Визуализация шкалы (`TimelineVisualBuilder`) использует `world.reference_point` как подпись метки нуля; заголовок маркерного события на главной линии **синхронизируется** с настройками мира при каждом сохранении (`WorldReferencePointSyncService`), чтобы не расходились текст на шкале и точка на линии.
- Линии и события: **модальные формы** и маршруты `timeline.lines.*`, `timeline.events.*` (создание, правка, удаление).
- **Контекстное меню** по правому клику на холсте: пустое место — создать линию; по линии — создать событие, редактировать линию, удалить линию (основную линию удалить нельзя); по точке события — редактировать / удалить событие (несколько событий в одном годе — отдельные пункты). Удаления с подтверждением.
- Линия, созданная из биографии, помечается **`source_biography_id`**; при удалении такой линии записи в биографии не трогаются.
- **Событие** может иметь флаг **`breaks_line`**: на второстепенных линиях линия на холсте обрывается на этом годе; на **главной** линии флаг недоступен и не применяется. Главная линия визуально всегда до конца холста.
- **Месяц и день** события на таймлайне — числа **1…100** (вымышленные календари), не привязаны к григорианской проверке дат.
- После добавления, правки или удаления события вызывается **`TimelineLine::recalculateBoundsFromEvents()`**: левая граница — `min(текущий start_year, min(годов событий))`, правая у ручных второстепенных — `max(текущий end_year, max(годов событий))`, чтобы период линии не сужался только до годов событий (событие в начале отрезка не обнуляет правый конец). У **главной линии мира** и у линий из **биографии** (`extends_to_canvas_end`) поле `end_year` остаётся **null** — линия визуально тянется до конца холста.

### Карточки (истории)

- У мира много **историй**; у истории — **карточки** (сценарный метод: абзацы, декомпозиция, порядок, подсветка и т.д.).
- Связь только через `world_id` / `story_id`; с таймлайном и биографиями на уровне БД **не связаны** (отдельные сюжетные артефакты).

### Биографии

- **Профили персонажей** и **события биографии** с датами на шкале мира (`epoch_year` и др.).
- **Рождение и смерть** в профиле задаются **числами**: год, опционально месяц и день (**1…100**), без календаря устройства — под вымышленную шкалу времени.
- Добавление событий в блоке «События» — **сворачиваемая форма** (аккордеон). У события можно отметить **«конец линии»** для таймлайна (`breaks_line` в БД); при выкладке на линию значение переносится в событие таймлайна (на основной линии мира не применяется).
- Связь с таймлайном:
  - можно создать **линию** по персонажу и выложить на неё события биографии;
  - можно отправить **одно** событие на выбранную линию таймлайна;
  - если линия персонажа уже создана на таймлайне, кнопка меняется на **«Удалить с таймлайна»** (линия и события на ней удаляются; записи в биографии остаются, можно вынести снова).
- Технически: `BiographyTimelineSyncService`, поле **`timeline_lines.source_biography_id`**, pivot `biography_timeline_event`, у события таймлайна при необходимости morph `source` на `BiographyEvent`.

### Бестиарий

- **Существа** мира, алфавитный указатель, поиск, PDF. Связь с миром через `world_id`; с таймлайном/биографиями **без** общей таблицы связей (пока).

### Доска связей

- Заглушка UI: `GET /worlds/{world}/connections`. Планируется единое полотно сущностей и связей между ними.

### Карты

- Маршруты `GET /worlds/{world}/maps`, `GET /worlds/{world}/maps/{map}`: полотно Konva, спрайты из `public/sprites`, объекты карты с координатами, названием и описанием (`world_map_sprites`).
- В разметке Noema ссылки **«Объекты карты»** (`module=1`) ведут только на объекты с **осмысленным названием**; пустое имя и служебная подпись вида «объект на карте #N» для ссылок не допускаются (не показываются в списке выбора и не проходят проверку при сохранении текста).

### Аккаунт

- **Профиль**: отображаемое имя, биография, аватар (файлы в каталоге пользователя).
- **Команда** и **глобальные настройки Noema**: интерфейс-заглушки без бэкенда.

### Правый верхний тулбар (иконки в шапке страницы)
