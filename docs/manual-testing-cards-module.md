# Ручное тестирование: модуль «Карточки»

Документ для проверки сценариев, которые **невозможно или нецелесообразно** полностью воспроизвести автотестами: визуальное поведение, браузеры, жесты, печать, доступность, UX при ошибках сети и т.д.

Автотесты покрывают HTTP-слой (маршруты, редиректы, валидация, часть JSON API, PDF как ответ с заголовками). Ниже — что должен пройти человек перед релизом или при существенных правках UI/JS.

---

## 1. Список историй (`/worlds/{world}/cards`)

- Отображение названий историй, циклов, превью/пустых состояний — **визуально** без обрезки длинных названий.
- Фильтр по циклу: переключение, сброс, поведение при отсутствии историй в выбранном цикле (пустой список, сообщения).
- Кнопка «Журнал» ведёт на журнал модуля карточек; иконка и подсказки читаемы.
- **Навигация с клавиатуры**: таб-порядок, фокус на кнопках и ссылках, Enter/Space на интерактивных элементах.
- **Мобильная ширина**: карточки/список не ломают вёрстку (320px / 375px / планшет).

---

## 2. Создание истории (форма на списке)

- Создание с именем, циклом, синопсисом; проверка **обрезки пробелов** в полях (как в коде — trim).
- После создания — переход на страницу истории, **четыре стартовые карточки** с ожидаемыми заголовками.
- Повторная отправка формы (двойной клик): нет дубликатов историй (по возможности проверить вручную).

---

## 3. Страница истории (`cards.show`): режим «модалка» и «страница»

- В настройках истории переключение **modal / page**: сохранение, перезагрузка страницы — режим сохранён.
- **Modal**: открытие карточки, закрытие по клику вне / Escape (если задумано), прокрутка длинного текста, фокус не «теряется» за модалкой.
- **Page**: переход по «Редактировать» открывает полноэкранное редактирование; возврат на ленту истории понятен пользователю.

---

## 4. Сетка карточек, порядок, подсветка

- **Перетаскивание** (если используется drag-and-drop): порядок меняется, после перезагрузки порядок сохранён; граничные случаи — одна карточка, много карточек (50+).
- Альтернатива/дополнение: любые **кнопки «вверх/вниз»** — те же проверки.
- **Подсветка** (pin): только одна карточка подсвечена; переключение между карточками; снятие подсветки.
- Проверка **на тач-устройстве** или эмуляторе: перетаскивание/клики без «залипания».

---

## 5. Редактирование содержимого (разметка Noema)

- Ввод текста, **жирный/курсив**, ссылки на сущности мира — превью и сохранённый вид совпадают с ожиданием.
- **Ссылки на объекты карты** (`module` «Объекты карты»): в модалке выбора в списке есть только объекты с **заданным названием**; объектов без названия и с служебной подписью «объект на карте #…» в списке **нет**. После сохранения карточки ссылка в тексте ведёт на превью с этим названием.
- Если в текст вручную вставить `[link module=1 entity=…]` на объект без подходящего названия — при сохранении **422** и сообщение об ошибке (не «тихий» успех).
- **Некорректная разметка**: ошибка показывается человекочитаемо (не только в консоли); при JSON-запросах — то же для фронта.
- **Вставка из буфера** (из Word, Google Docs, веб-страницы): нет поломки вёрстки; при необходимости — очистка лишнего HTML (если предусмотрено продуктом).
- Длинный текст: производительность прокрутки, отсутствие «подвисаний» при вводе.

---

## 6. Массовое добавление и декомпозиция

- **Массовое добавление** пустых карточек: число 1, граница 100, отказ при 0/101 — сообщения понятны.
- После добавления — **прокрутка к концу** списка (якорь/скролл), если это заявлено в UI.
- **Декомпозиция одной карточки** (≥2 абзацев): разбиение корректно; карточка с одним абзацем — понятное сообщение об ошибке.
- **Декомпозиция всех**: при отсутствии подходящих карточек — сообщение без падения страницы; при успехе — обновлённая сетка и скролл.

---

## 7. Удаление карточки и истории

- Удаление карточки **со страницы редактирования**: редирект «назад» не ведёт на «битую» страницу (пустая история / список).
- Удаление с опцией **вернуться к ленте истории** — фактическое поведение совпадает с подписью кнопки.
- Удаление **всей истории**: подтверждение; узлы на доске связей для карточек этой истории исчезают (регрессия интеграции со «Связями»).

---

## 8. PDF истории

- Автотест проверяет только **статус и заголовки ответа**. Вручную:
  - открыть PDF в Acrobat/Chrome: **кодировка кириллицы**, переносы, отсутствие «квадратиков»;
  - печать на A4: поля, обрезка;
  - имя файла при скачивании осмысленное (транслит/slug).

---

## 9. Журнал активности (модуль / история / карточка)

- Со страниц модуля карточек кнопка журнала ведёт на **узкий** журнал карточек мира.
- Очистка журнала с подтверждением; после очистки список пуст и нет 500-х ошибок.

---

## 10. Безопасность и доступ

- Пользователь **Б** не открывает карточки мира пользователя **А** (прямые URL с чужими id).
- Скрытый мир (`onoff = false`): те же URL — недоступны (404), без утечки названий в ответе.

---

## 11. Сеть и сессия

- Сохранение карточки при **обрыве сети** (DevTools → Offline): сообщение, данные не потеряны без ведома пользователя (или явная ошибка).
- **Истечение сессии** посреди редактирования: редирект на вход, после входа — предсказуемое поведение (потеря/сохранение черновика — по политике продукта).

---

## 12. Кросс-браузерность

- Минимум: **Chrome**, **Firefox**, **Safari** (или WebKit): ключевые сценарии из разделов 3–7.
- Проверка **масштаба страницы** 100% / 125%: модалки и сетка не разъезжаются.

---

## 13. Регрессия связанных экранов

- После изменений в `story-cards.js` / стилях: дашборд мира, таймлайн, связи — **косвенно** убедиться, что общий хедер/тема не сломаны.
