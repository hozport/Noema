  ## PHPDoc (PHP)

Для **каждого нового или существенно изменённого** класса и публичного/защищённого метода добавляй полный PHPDoc. Не дублируй очевидное в теле кода; смысл — в docblock.

### Классы

- Первая строка: короткое **название / роль** класса (как заголовок).
- Пустая строка.
- Абзац: **что делает** и при необходимости контекст (где используется, ограничения).

Пример:

```php
/**
 * Общий класс для работы с Баннерами
 *
 * Отвечает за базовые функции работы с баннерами,
 * на текущий момент актуально только для Главной
 */
```

### Методы

- Краткое описание назначения метода (одна или несколько строк).
- Пустая строка.
- `@access` со значением `public`, `protected` или `private` (как видимость метода).
- `@param` для каждого параметра: тип, имя, краткое описание.
- `@return` с типом; для `void` указывай `@return void`.

Пример:

```php
/**
 * Возвращает шапку и дополнительную навигацию на странице
 *
 * @access protected
 * @param string $title Название страницы
 * @param string $description Описание страницы
 * @return string
 */
```

Выравнивание звёздочек и отступов — как в существующем файле (обычно 5 пробелов перед `*` у методов внутри класса).

## Важные внутренние комментарии (`##`)

Для **заметных** пояснений внутри кода (нестандартная логика, предупреждение, граница крупного блока) используй комментарий, где текст **начинается с `##`** (два решётки и пробел).

В **PHP**:

```php
## Впереди важный блок!
```

В **JavaScript / TypeScript**:

```js
// Впереди важный блок!
```

Не смешивай с PHPDoc (`/** ... */`). Обычные пояснения без повышенной важности — обычный однострочный комментарий **без** префикса `##`.
