# Модуль «Таймлайн» — спецификация

Документ фиксирует согласованные решения по визуализации, данным и поведению (по состоянию на обсуждение с автором).

## Цели

- Единая временная ось мира: **год от точки отсчёта** (целое число, **≥ 0**).
- **Основная линия** истории мира — всегда узнаваема (бордо, толстая); дополнительные линии — тоньше, цвет задаёт автор.
- Холст **шире экрана** (минимум порядка 3000px по горизонтали в пустом состоянии); горизонтальная прокрутка общая для области линий и основной линии.
- **Разрывы** в отрисовке — для **пустот** между кластерами данных.
- **Зум + пан + агрегация** — для **плотных** данных на длинном отрезке без больших пустот.

## Хранение

- В БД — **абсолютные** значения времени; масштаб, разрывы, позиция на экране — **представление**.
- Для позиции на оси: **год от сотворения** (обязателен).
- Опционально: **точная дата** (день/месяц/год) для подсказок; **на линии** точка привязана к **году** (в тултипе может отображаться полная дата).

## Визуал

- Линии — не «полосы UI», а **геометрические линии** (SVG/Canvas): у основной **толщина ~10px**, у остальных меньше.
- Цвет основной линии — **строго бордо** (фиксированный токен темы).
- События — **круги** не меньше **`max(8px, толщина линии)`**, цвет совпадает с линией.
- Перекрытия нескольких событий в **одном году** на одной линии — **один маркер + счётчик** N.
- Подсказка при наведении (в интерактивной фазе): год, название; при наличии — точная дата.

## Граница времени и холста

- **Правый край холста** = визуальный конец поля времени.
- **Правая граница по времени:** `max(все события, все концы линий, при необходимости правая граница основной линии) + padding`.
- В **году 0** на **основной линии** по умолчанию — **точка отсчёта** (метка из настроек мира при создании).

## Масштаб без «заранее известной» длины истории

Система **не задаёт** фиксированный горизонт (вроде «всегда 3000 лет»). Длина оси **всегда выводится из данных**:

1. **Хранение** — по-прежнему абсолютные годы; **нижняя граница оси отображения** для мира — **0** (точка отсчёта).
2. **Верхняя граница** — `dataMax + padding`, где `dataMax` — максимум по всем событиям и концам линий, участвующим в оси.
3. **Padding** — доля от размаха данных (с нижним и верхним ограничением), чтобы справа оставался «воздух»; при очень короткой истории (например **всего 100 лет** событий 0…100) размах по времени ~100 лет, padding небольшой — **вся сотня лет растягивается на полную ширину холста** (линейное отображение `x = (year / tMax) * width`). То же 3000px ширины означают «пикселей на год» больше, чем при истории на тысячелетия — читабельность сохраняется.
4. **Пустой или почти пустой мир** — задаётся **минимальное окно** по времени (например 0…100 лет), чтобы ось не схлопнулась в точку.
5. **Деления шкалы (тики)** — **не фиксированные 500 лет**, а **шаг от размаха** `[0, tMax]`: для короткого отрезка — мелкий шаг (10–50 лет), для длинного — крупнее (100–500 лет), чтобы подписи не сливались.

Итог: история на **100 лет** и на **100 000 лет** обрабатываются одинаково по логике — **автоматический домен времени + линейная проекция на ширину холста**; отличаются только числа `tMax`, шаг тиков и плотность событий (зум/агрегация — на этапе интерактива).

## Создание линии (логика полей)

- Название, **год начала**, опционально **год конца**, **цвет**.
- Если конец не задан: линия до **визуального конца холста** (до правого края поля времени).

## Создание точки

- Линия, название, дата (год для оси; день/месяц опционально для подсказки), флаг **«обрывает линию»**.
- Если **обрывает**: после этой точки линия **не рисуется**.
- Продолжение после обрыва: **снять флаг** с обрывающей точки **или** добавить точку после без флага обрыва — линия снова идёт до конца времён (если нет более позднего обрыва). Детали приоритета — зафиксировать в UI при реализации форм.

## Раскладка

- Дополнительные линии: **снизу вверх**, шаг по вертикали **~25px** между линиями (между визуальными дорожками).
- **Основная линия** — у **нижнего края** рабочей области таймлайна; **по вертикали** всегда в зоне видимости layout’а; **по горизонтали** прокручивается **вместе** с широким холстом (один общий контейнер `overflow-x`).

## Текущая стадия реализации

- Данные в БД: таблицы `timeline_lines`, `timeline_events`; связь с биографиями — таблица `biography_timeline_event` (пока без UI).
- При создании мира и при миграции существующих миров без линий создаётся **основная линия** и **первая точка** (год 0, дата 01.01, заголовок из «Точки отсчёта» мира или «Точка отсчёта»).
- Холст строится из БД: `App\Support\TimelineVisualBuilder` (демо-класс `TimelineVisualDemo` остаётся для вспомогательных расчётов шкалы/отступов).
- Маршрут страницы: `GET /worlds/{world}/timeline`, имя **`worlds.timeline`** (`TimelineController@show`). CRUD линий и событий: `timeline.lines.*`, `timeline.events.*`.
- Создание линий и событий — модальные формы (верхнее меню и контекстное меню с предзаполнением года / линии).
- Пан/зум — позже.
