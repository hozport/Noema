# Ручное тестирование: модуль «Таймлайн»

Документ для проверки сценариев, которые **невозможно или нецелесообразно** полностью воспроизвести автотестами: визуальное поведение холста, браузеры, жесты, зум, перетаскивание линий, качество PDF и т.д.

Автотесты покрывают HTTP-слой (страница, PDF-ответ, линии, события, очистка, перестановка, права доступа), разметку шаблона PDF, вспомогательную логику дат/описаний. Ниже — что должен пройти человек перед релизом или при существенных правках UI/JS.

---

## 1. Страница таймлайна (`/worlds/{world}/timeline`)

- **Первый экран**: шапка и холст укладываются во вьюпорт без «лишней» прокрутки body; футер доступен при прокрутке страницы (если включён в вёрстке).
- **Масштаб (зум)**: колесо с модификатором / контролы зума меняют только содержимое холста; страница целиком не «раздувается» по высоте.
- **Панорама**: перетаскивание фона скроллит холст; перетаскивание **не** начинается с пустой области строки, если линия переставляемая — только с полосы линии или подписи (см. спецификацию UX).
- **Перестановка дополнительных линий**: вертикальный жест с полосы/подписи, предпросмотр обмена с соседом, сохранение порядка после перезагрузки.

---

## 2. События и линии (создание, редактирование, удаление)

- Создание **дополнительной линии** (имя, годы, цвет) — появляется на холсте в ожидаемом порядке.
- Создание **события** на линии (дата, название, опции вроде «ломает линию» для неосновной линии).
- Редактирование и удаление события и **удаление дополнительной линии** — данные на холсте и в формах согласованы.
- **Очистить таймлайн** (модалка): удаляются дополнительные линии и лишние события основной линии; маркер точки отсчёта сохраняется; после операции нет «битых» состояний.

---

## 3. Настройки с холста (модалка)

- Поля «инициирующее событие» и «ограничить таймлайн» сохраняются; при JSON-ответе холст обновляется без полной перезагрузки страницы (если так задумано).
- Ошибки валидации отображаются в модалке, форма не «молчит».

---

## 4. Контекстное меню и подсказки

- Правый клик по **точке** / **линии** / **фону** открывает соответствующее меню; пункты ведут на ожидаемые действия (редактирование, переходы к связанным сущностям — по продукту).
- Подсказки по точкам (тултипы) читаемы, не перекрывают критичные элементы.

---

## 5. Экспорт PDF

- Автотесты проверяют статус, `Content-Type`, сигнатуру PDF и содержимое **HTML-шаблона** документа. Вручную:
  - скачать PDF со страницы таймлайна: **кириллица** в заголовках и тексте, без «квадратиков»;
  - структура: «Таймлайн» → название и описание мира → **Основная временная линия** → события → затем **каждая дополнительная линия** с именем как заголовком раздела;
  - **отступы между разделами** визуально достаточные (заголовки не слипаются с предыдущим блоком);
  - для событий с привязкой к биографии/фракции — в PDF при наличии текста в источнике показывается **описание** (после очистки разметки);
  - имя файла при скачивании осмысленное (`timeline-…pdf`);
  - печать на A4: читаемость, переносы.

---

## 6. Журнал активности

- Кнопка «Журнал» на таймлайне ведёт на журнал с фильтром по модулю таймлайна; записи создаются при значимых действиях (создание линии/события, очистка и т.д. — по политике продукта).

---

## 7. Безопасность и доступ

- Пользователь **Б** не открывает таймлайн и PDF мира пользователя **А** (прямые URL).
- Скрытый мир (`onoff = false`): те же URL — **404**, без утечки названий.

---

## 8. Сеть и сессия

- Сохранение настроек таймлайна при **обрыве сети**: понятное сообщение об ошибке (или повтор запроса — по UX).
- **Истечение сессии** посреди работы: редирект на вход; после входа поведение предсказуемо.

---

## 9. Кросс-браузерность и устройства

- Минимум: **Chrome**, **Firefox** (и при возможности **Safari**): зум, панорама, модалки, PDF-ссылка.
- Узкая ширина / тач: холст прокручивается, кнопки доступны; при необходимости — проверка `touch-action` на зонах перестановки линий.

---

## 10. Регрессия соседних экранов

- После правок в `timeline.js` / стилях таймлайна: **дашборд мира**, **карточки**, **связи** — косвенно убедиться, что общий хедер и тема не сломаны.
