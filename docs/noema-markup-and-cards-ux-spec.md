# Спецификация: разметка Noema, редактор карточек и смежный UX

> Редактор разметки и API сущностей (`worlds.markup.entities`, `worlds.markup.resolve`) подключены к карточкам, бестиарию, биографиям и фракциям. Ниже — ориентир по UX; детали верстки см. в Blade и `resources/js/markup-editor.js`.

## 1. Модель разметки (сервер)

| Элемент | Синтаксис | Примечание |
|--------|-----------|------------|
| Жирный | `[b]…[/b]` | |
| Курсив | `[i]…[/i]` | |
| Подчёркивание | `[u]…[/u]` | |
| Зачёркивание | `[s]…[/s]` | |
| Ссылка на сущность | `[link module=M entity=E]…[/link]` | M, E — целые числа |
| Экранирование | `\` перед символом | |

- Валидация: см. `App\Markup\NoemaMarkupValidator` (например, перенос строки внутри тега — ошибка). При сохранении полей в контексте мира дополнительно проверяются ссылки `module=1` (объекты карты): объект должен существовать в этом мире и иметь **осмысленное название**; пустой `title` и служебная подпись вида «объект на карте #N» не допускаются.
- Рендер HTML/превью: парсер и рендереры в `App\Markup\`.

## 2. Клиент: поле разметки (`bindNoemaMarkupField` / `bindCardMarkupEditor`)

**Общие данные и состояние**

- Скрытое поле `content` (или аналог) синхронизируется с текстом редактора перед сабмитом (`syncBeforeSubmit`).
- Режимы: просмотр (отрендеренный HTML) и редактирование (CodeMirror или fallback `textarea`).
- Переход в редактирование: клик по области просмотра (pointerdown на форме).
- Выход в просмотр: клик вне `markupEditBoundary` (если задан) или клавиша **Esc** в режиме редактирования, с предварительным `syncBeforeSubmit`. Esc не срабатывает, пока открыты модалка ссылки или предпросмотр; `preventDefault`, чтобы не закрывался родительский `<dialog>` (модалка карточки).

**Панель форматирования**

- После выделения текста (`pointerup`): плавающая панель с теми же действиями, что и контекстное меню.
- Правый клок / `contextmenu`: то же меню в координатах курсора.
- Прокрутка скролла редактора скрывает панель.
- Горячие клавиши в CodeMirror: Mod+B / Mod+I / Mod+U; зачёркивание — **Mod+Shift+S** (не Mod+S — резерв под сохранение страницы в браузере).

**Модалка ссылок**

- Общий `#linkEntityModal` на страницу; выбор модуля и сущности; подтверждение оборачивает выделение в `[link …]…[/link]`.

**Связка с модулями**

- URL `worlds.markup.entities` и `worlds.markup.resolve` задаются в `data-markup-entities-url` / `data-markup-resolve-url` на формах карточек, бестиария, биографий и фракций.
- Список объектов карты (`module=1`) содержит только спрайты с названием, подходящим для ссылки (см. `WorldMapSprite::titleQualifiesForMarkupEntityLink`).

## 3. Модуль «Карточки»: модалка на сетке истории

**Данные формы**

- `title`, `content`, `_method=PUT`, `is_highlighted` (`0` / `1`).
- Закладка: только переключение скрытого поля до сохранения; на сервер уходит вместе с остальным.

**Черновик (sessionStorage)**

- Ключ: `wb:cardDraft:{worldId}:{storyId}:{cardId}`.
- Сохраняются только `title` и `content`; закладка не в черновике (берётся из DOM при открытии).

**Сохранение**

- AJAX: `fetch` с `Accept: application/json`, тело — `FormData` формы.
- Успех: очистка черновика, закрытие модалки, `location.reload()`.
- Ошибка 422: показ сообщения (например `alert`), форма открыта.

**Паритет с другими модулями**

- При добавлении новых модалок с разметкой — тот же контракт JSON/redirect (`Accept: application/json`, 422 с телом ошибок), чтобы не ломать навигацию.

## 4. Модуль «Карточки»: страница редактирования карточки

- Обычный POST/PUT формы; `is_highlighted` в скрытом поле; пин — «применится после Сохранить».
- Предупреждение при уходе по ссылке «Назад», если форма грязная.
- Черновик при загрузке страницы (тот же ключ).

## 5. Журнал активности модуля «Карточки»

- Маршрут: `cards.module.activity` — только записи, где `subject_type` — модель истории или карточки этого мира.
- Ссылка с главной списка историй: кнопка журнала ведёт на этот маршрут.
- Кнопка «Назад» на странице журнала: главная модуля карточек (`cards.index`), не дашборд мира.

**Прочие модули**

- У бестиария, биографий и фракций есть отдельные маршруты журналов с фильтром по типу субъекта; при изменении логики активности сверяйте с `ActivityLog` и маршрутами в `routes/web.php`.

## 6. Задания для ручного тестирования

### 6.1 Разметка в редакторе (страница карточки и модалка)

1. Открыть редактирование, ввести текст, переключить просмотр ↔ редактор кликом вне области и снова по **Esc** (модалка карточки не должна закрываться).
2. Выделить слово мышью: появляется панель; применить жирный — вокруг выделения появляются `[b]…[/b]`.
3. Двойной клик по слову → панель → курсив.
4. Правый клок: меню в точке курсора; ссылка на сущность открывает модалку и вставляет теги.
5. Проверить Mod+B, Mod+I, Mod+U, Mod+Shift+S с курсором и с выделением.
6. Прокрутить длинный текст в редакторе при открытой панели — панель исчезает.
7. Сохранить карточку: содержимое на сетке/странице совпадает с ожидаемым HTML.

### 6.2 Модалка на странице истории

1. Открыть историю, открыть карточку в модалке, изменить текст, закрыть по крестику с несохранёнными правками — должно быть предупреждение.
2. Включить/выключить закладку без сохранения — закрытие тоже считается грязным.
3. Сохранить: перезагрузка, данные и подсветка на сетке актуальны.
4. Намеренно ввести невалидную разметку (перенос внутри тега), сохранить — сообщение об ошибке, данные не потеряны.
5. Ссылка на объект карты без подходящего названия (вручную вставить `[link module=1 entity=…]` на такой объект) — при сохранении ошибка валидации.

### 6.3 Черновики

1. Правка в модалке, закрыть без сохранения, снова открыть ту же карточку — предложение восстановить черновик.
2. То же на странице карточки при первом заходе после правок.

### 6.4 Журнал «Карточки»

1. С главной «Карточки» открыть историю изменений — только события историй/карточек этого мира (не таймлайн, не карта и т.д.).
2. «Назад» ведёт на список историй модуля карточек.
3. Со страницы истории / карточки — узкие журналы ведут назад к ожидаемым экранам.

### 6.5 Регрессия других модулей

1. Бестиарий / биографии / фракции: сохранение полей с разметкой (если уже подключена) без JS-ошибок.
2. Таймлайн: журнал таймлайна открывается и «Назад» ведёт к таймлайну.
